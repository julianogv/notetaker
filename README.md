# Notetaker

A command-line tool that records meetings, transcribes locally with Whisper, and
generates a structured summary (discussion points, decisions, action items, and
notes) via an LLM CLI of your choice (kiro-cli, claude-code, etc).

Notetaker exists to reduce the manual effort of taking notes and distributing
meeting summaries while keeping audio 100% local and sending only text to the LLM.

## Language

**Meeting**:
A recorded session, from `start` to `stop`. Each Meeting corresponds to a folder
on disk with all its files.
_Avoid_: session, call, recording

**Track**:
An audio stream recorded separately. In online mode there are two: `mic`
(your voice) and `system` (participants' voices, captured from audio output monitor).
In in-person mode there is only `mic`.
_Avoid_: channel, stream, lane

**Mode**:
How the audio is captured. `online` records mic + system into separate Tracks;
`in-person` records only the mic; `listener` records only the system (you
listening in on an online meeting).
_Avoid_: meeting type

**Diarization**:
Attribute speech segments to who spoke. `level1` distinguishes you (Track mic) from
participants (Track system) without ML. `level2` uses ML to separate each speaker.
_Avoid_: speaker separation, speaker split

**Summary**:
The output generated by the LLM from the transcript, in structured Markdown.
Persisted as `summary.md`.
_Avoid_: minutes, report, digest

**LLM Provider**:
Which external CLI to use to generate the Summary from the transcript: `kiro`
(kiro-cli chat --no-interactive) or `claude` (claude -p). Configured in
`[llm].provider`; the transcript goes via stdin. Notetaker does not talk to
LLM APIs directly.
_Avoid_: model, AI, backend

**Meeting Language**:
The language spoken in the meeting (`auto`, `pt`, `es`, `en`). Determines the
transcript language and, by default, the Summary language.
_Avoid_: locale

## Installation

### Option A — Automatic script (recommended)

The `notetaker.sh` (Linux/macOS) and `notetaker.bat` (Windows) scripts handle
everything: install `uv` (if needed), check `ffmpeg`, create the virtual
environment, and run the application.

```bash
# Linux/macOS
./notetaker.sh --setup            # set up the environment
./notetaker.sh start "my meeting"
./notetaker.sh --help             # full script help
```

```bat
REM Windows (CMD or PowerShell)
notetaker.bat --setup
notetaker.bat start "my meeting"
notetaker.bat --help
```

### Option B — Manual installation

```bash
# 1. System dependency for audio capture
sudo apt install ffmpeg                       # Linux (Debian/Ubuntu)
brew install ffmpeg                           # macOS
winget install Gyan.FFmpeg                    # Windows (or: choco install ffmpeg)

# 2. Install notetaker (Python >= 3.11)
pip install -e .

# 3. (Optional) Level 2 diarization via ML
pip install -e ".[diarization]"

# 4. (Optional) NVIDIA GPU acceleration (CUDA)
pip install -e ".[gpu]"
```

In both cases, you need an LLM CLI already installed and authenticated
(e.g., `claude`, `kiro-cli`). Notetaker just invokes it; it does not manage keys.

## Platforms

### Linux
Capture via PulseAudio/PipeWire. Online mode (mic + participants) works
natively: audio output is captured from the sink monitor. No extra installation
beyond `ffmpeg`.

### macOS
Capture via AVFoundation. **In-person mode** (mic only) works directly.

**Online mode** (capturing participants' voices) requires a virtual audio device,
since macOS does not expose audio output for capture natively:

1. Install [BlackHole](https://existential.audio/blackhole/) (free):
   `brew install blackhole-2ch`
2. In *Audio MIDI Setup*, create an **Aggregate Device** with BlackHole + your
   normal output (to listen and capture at the same time).
3. During the meeting, use this device as audio output.

Notetaker detects BlackHole automatically at `start`. Run
`./notetaker.sh devices` to see what was detected. Without BlackHole, online mode
displays instructions and suggests `--mode in-person`.

> Note: on macOS, transcription runs on CPU (CTranslate2 does not support
> Metal/MPS). On Apple Silicon the CPU is fast enough for smaller models.

### Windows
Capture via DirectShow (dshow). **In-person mode** (mic only) works directly —
Notetaker uses the detected default input device.

**Online mode** (capturing participants' voices) requires a loopback audio device,
since DirectShow does not expose audio output for capture natively:

1. **Stereo Mix** (simpler if your card supports it): enable it in *Sound Settings > Recording*
   (right-click the list and check "Show disabled devices"), then enable it.
2. **Virtual device** (universal): install [VB-CABLE](https://vb-audio.com/Cable/)
   or VoiceMeeter and route audio output to it during the meeting.

Notetaker detects Stereo Mix / virtual device automatically at `start`. Run
`notetaker.bat devices` to see what was detected. Without a loopback, online mode
displays instructions and suggests `--mode in-person`.

> Note: on Windows, transcription uses NVIDIA GPU (CUDA) when available; otherwise
> CPU. Also check microphone permissions for your terminal app in
> *Settings > Privacy & Security > Microphone*.

### GPU Acceleration (NVIDIA)
When an NVIDIA GPU is detected, transcription uses CUDA automatically
(much faster). Install the libs with `pip install -e ".[gpu]"` or
`./notetaker.sh` (which already syncs the environment). Without GPU, CPU is used normally.
Check the device in use via metrics displayed during processing.

## Configuration

On first run, Notetaker detects no config exists and offers to run an interactive
setup wizard, which asks for each option with a default value (press Enter to accept).
You can also run the wizard anytime:

```bash
./notetaker.sh setup
```

The wizard saves `~/.config/notetaker/config.toml`. If you prefer to edit the
file manually, it has this format:

```toml
storage_root = "~/notetaker"   # where Meetings are saved

[audio]
mic_source = ""        # empty = auto (PulseAudio default source)
monitor_source = ""    # empty = auto (monitor of default sink)

[whisper]
model = "medium"       # tiny | base | small | medium | large-v3
language = "auto"      # auto | pt | es | en

[summary]
language = "meeting"   # "meeting" = meeting language | pt | es | en

[llm]
provider = "kiro"      # kiro | claude
```

Tips:
- **Devices**: leave empty for automatic detection. When switching headsets/speakers,
  Notetaker uses the active device at the time of `start`.
- **Whisper model**: `medium` gives good quality in Portuguese; `base`/`small`
  are faster on CPU; `large-v3` is most accurate (and slowest).
- **LLM**: any CLI that reads stdin and prints output works. Notetaker requests
  the summary delimited by sentinels, so it tolerates banners/ANSI in output.

## Usage

The examples below use `notetaker.sh` (recommended): it keeps the environment
updated and passes arguments to Notetaker. If you installed manually
(Option B), just replace `./notetaker.sh` with `notetaker` in any command.

See full help anytime:

```bash
./notetaker.sh --help
```

### Record a meeting (default mode, with live tracking)

```bash
./notetaker.sh start "sprint planning"
```

During recording, a live line shows elapsed time and audio size:

```
⠹ recording  12:34  audio: 3.2 MB  (Ctrl+C to stop)
```

Press **Ctrl+C** to stop. Notetaker then transcribes and generates the
summary, showing progress for each phase:

```
⠧ transcribing your voice (mic)...
⠴ transcribing participants (system)...
⠋ generating summary with LLM...
summary ready: ~/notetaker/2026-07-02_1234_sprint-planning/summary.md
```

### In-person meeting (microphone only)

```bash
./notetaker.sh start "client meeting" --mode in-person
```

### Choose language and diarization

```bash
# Meeting in English, summary in Portuguese, identify each speaker (ML)
./notetaker.sh start "weekly" --lang en --output-lang pt --diarization level2
```

### Detached mode (no live tracking)

Useful for automation or when you don't want the terminal tied up:

```bash
./notetaker.sh start "call" --no-watch   # returns immediately
# ... meeting happens ...
./notetaker.sh stop                      # stop and process in background
./notetaker.sh status                    # check progress
```

### Other commands

```bash
./notetaker.sh status                 # state of the most recent meeting
./notetaker.sh list                   # list all recorded meetings
./notetaker.sh devices                # show detected audio devices
./notetaker.sh summarize <folder>     # regenerate summary without re-transcribing
./notetaker.sh summarize <folder> --output-lang en   # ... in another language
```

## Files generated per meeting

Each Meeting is a folder in `storage_root`, named `YYYY-MM-DD_HHMM_title`:

```
2026-07-02_1234_sprint-planning/
├── audio-mic.opus        # your voice (Track mic)
├── audio-system.opus     # participants' voices (Track system; online only)
├── transcript-mic.txt    # transcript of your speech
├── transcript-system.txt # transcript of participants
├── transcript-full.txt   # combined transcript, labeled by speaker
├── summary.md            # the final Summary (Markdown)
└── meta.json             # session metadata
```

## Command reference

Via script (recommended) or, if installed manually, replace `./notetaker.sh`
with `notetaker`:

```
./notetaker.sh --setup            # install/update dependencies and exit
./notetaker.sh --help             # script help

./notetaker.sh start "title" [--mode online|in-person|listener] [--lang auto|pt|es|en] \
                             [--diarization level1|level2] [--output-lang meeting|pt|es|en] \
                             [--no-watch]
./notetaker.sh stop [--wait]
./notetaker.sh status
./notetaker.sh list
./notetaker.sh devices
./notetaker.sh summarize <folder> [--output-lang meeting|pt|es|en]
```

## Privacy

Audio never leaves your machine: transcription runs locally with Whisper.
Only the transcript text is sent to the configured LLM Provider. See
`docs/adr/0002-audio-local-text-remote.md` for details.
